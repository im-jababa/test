# 메모 프로그램 (테스트용)

간단한 메모 앱으로, 실행 중인 동안만 메모가 유지됩니다. 별도의 영구 저장은 없습니다.

**개요**

- 각 메모는 `id`, `제목`, `내용`, `생성 시각`, `마지막 변경 시각`을 가집니다.
- 프로그램 시작 시 메모 목록과 메모 추가 버튼이 표시됩니다.
- 메모가 존재하면 목록에서 메모를 눌러 상세 화면으로 이동해 제목과 내용을 입력/수정할 수 있습니다.
- 메모 추가 시 제목과 내용을 입력하고 생성합니다.
- 저장 버튼은 없으며, 입력 중 실시간으로 저장됩니다. 메모 추가 시에도 마찬가지입니다.
- 메모 목록에서 정렬 옵션을 선택해 `생성 시각 오름/내림`, `마지막 변경 시각 오름/내림`으로 정렬할 수 있습니다.
- 메모는 목록 화면과 메모 상세 화면에서 모두 삭제할 수 있습니다.

**프로젝트 구조**

- 최상위는 `main.rs`, `lib.rs`, `core/`, `feature/`로 구성합니다.
- `core/`는 기능(도메인)과 무관하게 공통으로 쓰이는 모듈이 위치합니다.
- `feature/`는 기능(도메인)별 하위 폴더를 둡니다.
- 각 feature는 폴더이며 내부에 다음을 포함합니다.
  - `domain.rs`: 해당 feature의 도메인 객체 정의
  - `id.rs`: 도메인 고유 아이디 타입 정의 (`type ID = ...`)
- 특정 도메인 하위 기능이 있으면 해당 도메인 폴더 안에 추가 폴더를 두어 구현합니다.
- 간단한 타입은 바로 모듈 파일을 둡니다.
  - 예: 사용자 권한 `feature/user/role.rs` + `mod.rs`에서 `mod role; pub use role::*;`
    - `feature::user::Role`로 사용 가능
- 더 큰 하위 개념은 하위 폴더를 두고 동일한 구조를 따릅니다.
  - 예: 사용자 그룹 `feature/user/group/` 하위에 `domain.rs`, `id.rs` 등 배치
- 네임스페이스를 최대한 활용해 짧고 일관된 경로를 사용합니다.
  - 예: `crate::feature::user::UserID`보다 `crate::feature::user::ID`
- `domain.rs`에 정의된 도메인 구조체는 상위에서 바로 쓰일 수 있도록 export합니다.
  - 예: `crate::feature::user::User`보다 `crate::feature::User`

**창 처리**

- UI 프레임워크는 `iced`를 사용합니다.
- 단일 창에서 좌측 메모 목록 / 우측 메모 상세의 분할 레이아웃으로 구성합니다.
- 초기 창 크기는 `900x600`, 리사이즈 가능으로 설정합니다.
- 선택된 메모가 없을 때 상세 영역에는 안내 문구(예: "메모를 선택하세요")를 표시합니다.

**추가 결정 사항**

- 대상 OS: Windows/macOS/Linux
- 메모 ID 생성: 증가형 `u64`
- 실시간 저장: 입력 이벤트 즉시 상태 반영
- `updated_at` 갱신: 제목/내용 변경 즉시 갱신
- 기본 정렬: `updated_at` 내림차순, 동률 시 `created_at` 내림차순
- 삭제 UX: 확인 다이얼로그 사용
- 생성 직후: 새 메모 자동 선택 + 제목 입력 포커스
- 삭제 직후: 선택 해제(상세 영역 빈 상태)
